Arial;mso-no-proof:yes'>For model training, we split the data into three parts,
one part which comprises 75 percent of the data, which is used for training.
Another part which comprises 12.5 percent of the data, which is used for validation.
The last part comprises 12.5 percent of data, which is used for testing. The
validation is used for selecting the best variation of a single model, e.g.
which K value to use for KNN and the test data is used to compare the
performances of different models e.g. ANN performance vs KNN performance. <o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:

is also used if we want to have a more general model, but in our case, it's
mostly unnecessary. <o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:

one is Min-Max scaling and the other one is Z-Score Normalization. While Z-Score
Normalization improves the performance of the ANN model, from a validation
accuracy of 0.836 to 0.86, which was trained on the 500 data points and
evaluated on 250 data points, it decreases the validation accuracy of KNN model
on the same data, from 0.872 to 0.8, therefore, all in all, it's not worth it
to use Z-Score Normalization.<o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:

Normalization, since Z-Score Normalization is usually used when there is no
well-defined range for a feature, whereas in our case there is a defined range
of [0,255] for our features which are G values for every pixel of image, since
images are all single colored(green) and therefore don't have a R or B values. <o:p></o:p></span></p>

predict function of the KNeighborsClassifier. Since learning in the K-NN
algorithm consists of storing data, in the fit function we store the data and
its labels in the class. In the predict function, for each inputted datapoint,
we use the Euclidean distance to find the closest K data points and assign it
the most frequent label in the found data. We also implement a gridsearch,
which is essentially a for loop, to find the best value for K in the range of
[1,10]. The reason we chose ten as the maximum value of K is that we have ten
labels or classes, therefore inspecting the ten nearest data points is a good
enough sample to detect the label of the data in question. The K-NN model was
trained on 20000 pictures of the training data and evaluated on 10000 pictures
of the evaluation data.<o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:
embed'><b><span style='font-size:14.0pt;line-height:115%;mso-bidi-font-family:
Arial;mso-no-proof:yes'>Implementation<o:p></o:p></span></b></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:
embed'><span style='font-size:14.0pt;line-height:115%;mso-bidi-font-family:
Arial;mso-no-proof:yes'>In this exercise, we had to implement the gridsearch to
find the optimum parameters for our ANN model. These parameters include the
number of hidden layers, amount of neurons in each hidden layer, and the
learning rate.<o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:
embed'><span style='font-size:14.0pt;line-height:115%;mso-bidi-font-family:
Arial;mso-no-proof:yes'>In our gridsearch, the minimum number of layers was one
and the maximum number of layers was two. Based on the universal approximation
theorem, at least one hidden layer is needed to be able to approximate any
function. As for why two is the maximum number of layers, by experimenting, it
was seen that after two layers, the performance doesn't improve by a noticeable
amount and for further performance improvement, it's better to use a different
network architecture like CNN. <o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:
embed'><span style='font-size:14.0pt;line-height:115%;mso-bidi-font-family:
Arial;mso-no-proof:yes'>At each layer, we could have either 200, 300, or 400
neurons. Even though it might be seen that it's better to have 784 neurons at
each layer, which is the size of the first layer, it's not as all pixels of the
input images hold relevant information for classification and it's better to move
the data to a space with fewer dimensions, e.g 400, 300 or 200, by setting the
number of hidden layers to 400, 300 or 200. Of course, we could have also used
a method to reduce the dimensions of our data as well and therefore not have
784 neurons at the first layer. <o:p></o:p></span></p>

<p class=MsoNormal dir=LTR style='text-align:justify;direction:ltr;unicode-bidi:
embed'><span style='font-size:14.0pt;line-height:115%;mso-bidi-font-family:
Arial;mso-no-proof:yes'>In our gridsearch we tried different learning rates of
0.0001 and 0.001, which are common values for learning rates in the industry.
As you can see learning rates are a value between zero and one and are inverse
powers of ten, which is a common practice. Both 0.00001 and 0.01 were omitted
because of their lack of efficiency, as<span style='mso-spacerun:yes'> 
</span>0.00001 was too low and therefore slowed down the learning and 0.01 was
too high and caused the learning process to be choppy. We could have also used
a method to dynamically change the learning rate, rather than have a fixed
learning rate, but it was mostly unnecessary and was not required in the
assignment. The ANN model was trained on 20000 pictures of the training data
and evaluated on 10000 pictures of the evaluation data.<o:p></o:p></span></p>

Arial;mso-no-proof:yes'>We can see there is a general downtrend in loss

by the network, however, with additional examples, classification for
distributions that are different from the norm are also learned, which appear
less and have less impact in decreasing the loss function value. <o:p></o:p></span></p>
